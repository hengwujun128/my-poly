# 08 - 踩坑记录

记录开发中遇到的典型问题、根因与修复，供排查复用。

## 1. git 提交时刷出大量 `npm warn Unknown project config`

### 现象

执行 `git commit` 时终端输出：

```
npm warn Unknown project config "strict-peer-dependencies". This will stop working in the next major version of npm.
npm warn Unknown project config "auto-install-peers". ...
npm warn Unknown project config "shamefully-hoist". ...
npm warn Unknown project config "ignore-workspace-root-check". ...
npm warn Unknown project config "install-workspace-root". ...
```

不影响提交结果，但每次提交都刷屏。

### 根因

- 这些键是 **pnpm 专属配置**，写在项目 `.npmrc` 里，对 pnpm 完全合法。
- 但 husky 钩子里用的是 `npx`（走 **npm**）：
  - `.husky/pre-commit` → `npx lint-staged --allow-empty`
  - `.husky/commit-msg` → `npx --no-install commitlint --edit "$1"`
- `npx` 触发 npm 读取 `.npmrc`，npm 不认识这些 pnpm 键，于是逐条告警。

### 修复

钩子改用 `pnpm exec`（本项目是 pnpm-only，直接跑本地二进制，不经过 npm）：

```sh
# .husky/pre-commit
pnpm exec lint-staged --allow-empty

# .husky/commit-msg
pnpm exec commitlint --edit "$1"
```

### 说明

- **不要动 `.npmrc`**：这些配置对 pnpm 是正确且必要的。
- 若将来确需用 npm 跑某些命令，可把 pnpm 专属键迁到 `.pnpmrc`；当前纯 pnpm 无此需要。
- 通用原则：本项目所有脚本/钩子统一用 `pnpm` / `pnpm exec`，禁止 `npm` / `yarn` / `npx`。

## 2. Cursor 规则 frontmatter 失效（无 globs / description）

### 现象

`.cursor/rules/*.mdc` 里配置了 `globs` / `description`，但作用域不生效（要么不加载，要么每轮都加载）。

### 根因

frontmatter 被写在**文件末尾**，且文件顶部还有编辑器插件自动注入的 HTML 注释头（`@Author / @Date ...`）。Cursor 要求 frontmatter 必须位于**文件第 1 行**的 `---` 之间。

### 修复

- 把 `---` frontmatter 块移到文件最顶部，删除顶部的注释头。
- 若编辑器插件会自动回填文件头注释，给 `.mdc` 关闭该注入，避免再次顶坏 frontmatter。

### 规则成本约定（省 token）

- `alwaysApply: true` 全文每轮注入，**只放红线**（技术栈锁定 + 禁用清单），保持精简。
- 细则用 `globs` 作用域化（如 `src/**/*.vue`、`src/api/**`），仅在相关文件在上下文时加载。
- 大段情境知识放 **skill**（仅 description 常驻，正文按需）。
