# 07 - 发布部署（小程序 + 后端）

## 架构一览

```
微信小程序（正式版/体验版）
        │ HTTPS
        ▼
https://api.zequan.club/v1   ← Nginx 反代
        │
        ▼
nest-prisma-api (PM2, :3000)
        ├── MySQL
        └── Redis
```

| 环境 | 小程序构建命令 | API 地址 |
|------|----------------|----------|
| 本地开发 | `pnpm dev:mp` | `http://127.0.0.1:3000/v1` |
| 体验版/正式版 | `pnpm build:mp:prod` | `https://api.zequan.club/v1` |

小程序按微信 `envVersion` 自动选 API（见 `src/utils/index.ts` → `getEnvBaseUrl()`）：

| 微信版本 | 环境变量 | 生产包默认值 |
|----------|----------|--------------|
| develop 开发版 | `VITE_SERVER_BASEURL__WEIXIN_DEVELOP` | 回退 `VITE_SERVER_BASEURL`（线上） |
| trial 体验版 | `VITE_SERVER_BASEURL__WEIXIN_TRIAL` | `https://api.zequan.club/v1` |
| release 正式版 | `VITE_SERVER_BASEURL__WEIXIN_RELEASE` | `https://api.zequan.club/v1` |

> 本地开发包（`pnpm dev:mp`）的开发版仍走 `127.0.0.1`，与线上发布包互不影响。

## 一、后端发布（nest-prisma-api）

线上 API 已部署在 `https://api.zequan.club`，Nginx 配置见 `nest-prisma-api/nginx/api.zequan.club.conf`。

### 方式 A：GitHub Actions 自动部署（推荐）

推送到 `main` 分支后自动部署到服务器。

**需在 GitHub 仓库配置：**

| 类型 | 名称 | 说明 |
|------|------|------|
| Secret | `SERVER_IP` / `SERVER_NAME` / `SSH_PRIVATE_KEY` | SSH 连接 |
| Secret | `WECHAT_MINI_APPSECRET` | 小程序 AppSecret |
| Variable | `DATABASE_URL` / `SHADOW_DATABASE_URL` | 数据库 |
| Variable | `NODE_REDIS_*` | Redis |

### 方式 B：服务器手动部署

```bash
ssh 你的服务器
cd /root/www/nest-admin-platform/nest-prisma-api

# 确保 .env 已配置（参考 .env.example）
# WECHAT_MINI_APPID=wx41143627eaf3541f
# WECHAT_MINI_APPSECRET=***

bash scripts/deploy-production.sh
```

验证：

```bash
curl https://api.zequan.club/v1/captchaImage
# 应返回 code:200 和验证码 SVG
```

## 二、小程序发布（my-poly）

### 1. 发布前检查

```bash
cd my-poly
pnpm release:check
```

### 2. 构建生产包

```bash
pnpm release:mp
# 产物：dist/build/mp-weixin
```

### 3. 上传代码

**方式 A：微信开发者工具（简单）**

1. 打开微信开发者工具
2. 导入项目：`dist/build/mp-weixin`（注意不是 dev 目录）
3. 点击「上传」→ 填写版本号与备注

**方式 B：CLI 自动上传**

```bash
# 需先在项目根目录放置 private.wx41143627eaf3541f.key（微信公众平台下载）
pnpm upload:mp
```

### 4. 微信公众平台配置（必做）

登录 [微信公众平台](https://mp.weixin.qq.com) → **开发 → 开发管理 → 开发设置**：

| 配置项 | 值 |
|--------|-----|
| request 合法域名 | `https://api.zequan.club` |
| uploadFile 合法域名 | `https://api.zequan.club`（若需上传） |

> 域名必须 **HTTPS**，且已在 ICP 备案。本地 `127.0.0.1` 不能用于正式版。

### 5. 提交审核与发布

1. **管理 → 版本管理**
2. 开发版本 → **选为体验版**（先内测）
3. 体验版验证登录、微信登录正常后 → **提交审核**
4. 审核通过 → **发布**（此时为正式版，走 `WEIXIN_RELEASE` 线上 API）

## 四、常见问题

### 正式版仍请求 127.0.0.1

- 是否用了 `dev:mp` 的包上传？必须用 `pnpm release:mp` 或 `upload:mp`
- 检查 `env/.env.production` 中 `VITE_SERVER_BASEURL` 是否为线上地址

### request:fail url not in domain list

- 公众平台未配置 `https://api.zequan.club`
- 或用了 HTTP 而非 HTTPS

### 微信登录失败

- 服务器 `WECHAT_MINI_APPSECRET` 是否与 AppID `wx41143627eaf3541f` 匹配
- 后端是否已部署最新 wxLogin 代码

### Failed to fetch / TypeError: Failed to fetch

微信开发者工具或真机里接口完全连不上时常见。

**1. 确认当前用的包和 API 地址**

控制台看启动日志：

```text
[my-poly] API baseUrl = ... | envVersion = develop/trial/release
```

| 场景 | baseUrl 应为 |
|------|----------------|
| `pnpm dev:mp` 开发版 | 默认 `http://127.0.0.1:3000/v1` |
| `pnpm release:mp` 上传的包 | `https://api.zequan.club/v1` |

**2. 开发版连 127.0.0.1 失败（最常见）**

- 本机要先跑：`cd nest-prisma-api && pnpm start:dev`
- **真机预览/真机调试** 不能访问电脑的 127.0.0.1，需改 `env/.env.development`：

```env
# 真机调试时改用线上（或改成电脑局域网 IP:3000）
VITE_SERVER_BASEURL__WEIXIN_DEVELOP = 'https://api.zequan.club/v1'
```

改完重新 `pnpm dev:mp`。

**3. 线上域名未配置**

公众平台 → 开发 → 开发管理 → 开发设置 → **服务器域名**：

- request 合法域名：`https://api.zequan.club`（不要带 `/v1`）

开发阶段可在微信开发者工具：**详情 → 本地设置 → 不校验合法域名**（仅开发工具内有效）。

**4. 自检命令**

```bash
curl https://api.zequan.club/v1/captchaImage
# 应返回 code:200
```


```bash
pm2 logs nestPrismaApi
pm2 list
curl http://127.0.0.1:3000/v1/captchaImage
```
