# 03 - 登录鉴权

## 架构

```
页面跳转 → routeInterceptor → 登录策略判断 → login.vue / wxLogin
                ↓
         tokenStore (Pinia持久化)
                ↓
         http interceptor 附加 Bearer Token
```

## 登录策略（当前配置）

**白名单策略**（`DEFAULT_NEED_LOGIN`）：默认所有页面需登录，仅以下页面免登录：

- `/pages/auth/login`
- `/pages/auth/register`

未登录访问其他页面 → 自动跳转登录页。

## 登录方式

| 方式 | 入口 | 说明 |
|------|------|------|
| 账号密码 | 登录页 | admin / admin123456 + 图形验证码 |
| 微信一键登录 | 登录页（仅 MP） | 授权头像昵称 + openId；未绑定返回 code `460` |
| 本机号码一键登录 | 登录页（仅 MP） | 微信手机号组件；未绑定返回 code `462` |

> 三种方式可独立使用。微信/手机号提示「需绑定」后，再用账号密码登录会附带 `wxCode` / `phoneCode` 完成绑定。

> 微信头像需用户授权 `getUserProfile`，并在小程序后台配置 download 合法域名 `https://thirdwx.qlogo.cn`。

> 微信登录走服务端 code2session，**本地 API（127.0.0.1）和线上 API 均可**，与页面重定向无关。

## 验证码说明

后端 `/captchaImage` 返回的是 **SVG 字符串**，不是 GIF base64。  
小程序优先将 SVG 写入本地临时文件再展示（比 data URI 更稳定），见 `src/utils/captcha.ts`。

测试账号：`admin` / `admin123456`

## 微信绑定与登录流程

### 首次使用（推荐）

```
1. 点击「微信一键登录」
   → openId 未绑定 → 返回 code=460，页面提示去账号密码登录

2. 填写 admin / admin123456 + 验证码，点击「登录」
   → 小程序自动 uni.login 获取 wxCode
   → POST /v1/login { username, password, uuid, code, wxCode }
   → 后端校验账号 → 将 openId 写入 sys_user.openId → 返回 JWT

3. 下次可直接「微信一键登录」
   → POST /v1/wxLogin { code }
   → 按 openId 找到 admin 用户 → 返回 JWT
```

### 已登录用户补绑

```
POST /v1/wxBind { code }   // 需 Bearer Token
```

### 后端配置

`nest-prisma-api/.env`：

```
WECHAT_MINI_APPID=wx41143627eaf3541f
WECHAT_MINI_APPSECRET=***
# 可选：C 端自动注册新用户（管理后台建议不设或 false）
# WECHAT_MINI_AUTO_REGISTER=true
```

| 错误码 | 含义 |
|--------|------|
| 460 | 微信未绑定系统账号 |
| 461 | 微信/账号绑定冲突（已绑其他） |

### 历史数据清理

若早期版本已自动创建 `wx_xxx` 用户，需先在数据库清除该用户的 `openId`，再按上述流程将微信绑定到 `admin` 等正式账号。

## 难点

- 小程序默认不走登录拦截，已通过 `LOGIN_PAGE_ENABLE_IN_MP = true` 开启
- `toLoginPage.ts` 原指向旧路径，已修正为 `/pages/auth/login`
- token 过期：单 token 模式下 401 直接跳登录页（无 refresh）

## 易错点

1. **验证码过期**：登录失败需刷新验证码（登录页已实现）
2. **redirect 编码**：路由拦截器用 `encodeURIComponent`，登录页 `onLoad` 需 `decodeURIComponent`
3. **persist 缓存**：调试时清缓存：微信开发者工具 → 清缓存 → 全部清除
4. **Bearer 前缀**：拦截器已加，后端 `logout` 用 `authorization.slice(7)` 取值

## 注意点

- 演示账号请向后端确认（默认若依系一般为 `admin/admin123`）
- 白名单/黑名单页面在 `EXCLUDE_LOGIN_PATH_LIST` 或页面 `definePage({ excludeLoginPath: true })` 配置

## 适用场景

- 新增需登录页面：白名单策略加 `excludeLoginPath`，黑名单策略不加
- 接入微信一键登录：先完成后端 `wxLogin`，再调 `tokenStore.wxLogin()`
