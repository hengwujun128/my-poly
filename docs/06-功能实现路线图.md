# 06 - 功能实现路线图

## 已完成（Phase 0）

- [x] unibest 基础模板（Vue3 + Vite5 + UnoCSS + pnpm）
- [x] Wot UI v2 集成
- [x] 登录策略 + 登录页（验证码）
- [x] lime-echart 图表 demo
- [x] API 对接 nest-prisma-api（login/getInfo/logout/captcha）
- [x] 环境变量（本地/线上）
- [x] 小程序 AppID 配置
- [x] wot-ui-v2 Cursor Skill

## 建议实施步骤

### Phase 1 - 基础能力（1~2 天）

1. 启动本地 `nest-prisma-api`，验证登录流程
2. 完善「我的」页面 UI（Wot UI 头像、昵称、退出）
3. 配置微信合法域名（上线前）

### Phase 2 - 微信登录（依赖后端）

1. 后端实现 `POST /v1/wxLogin`（code2session + 用户绑定）
2. 前端 `me.vue` 微信一键登录联调
3. 小程序 `manifest` 配置 scope（用户信息等）

### Phase 3 - 业务模块

1. 按后端模块拆分 `src/api/`（system、monitor 等）
2. 列表页 + z-paging 分页
3. 图表页接入真实业务数据

### Phase 4 - 发布

1. `pnpm build:mp:prod`
2. 微信开发者工具上传
3. 提交审核

## 各阶段难点汇总

| 阶段 | 难点 | 规避方式 |
|------|------|----------|
| 环境 | 小程序本地域名 | 127.0.0.1 + 关闭域名校验 |
| API | 字段命名差异 | 统一在 api 层 map |
| 登录 | wxLogin 未实现 | 先用账号密码，后端补齐后再切 |
| UI | v1/v2 混用 | 只用 @wot-ui/ui |
| 图表 | 包体积 | 按需引入 echarts 模块 |
| 发布 | 合法域名 / HTTPS | 提前在微信公众平台配置 |
