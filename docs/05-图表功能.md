# 05 - 图表功能（lime-echart）

## 已集成

通过 `pnpm create unibest add lime-echart` 注入：

- 组件：`src/uni_modules/lime-echart/`
- 示例：`src/pages-demo/lime-echarts/`
- Hook：`src/pages-demo/hooks/useEcharts.ts`
- 依赖：`echarts`、`lodash-es`

## 查看示例

1. `pnpm dev:mp` 或 `pnpm dev:h5`
2. 访问分包页面 `pages-demo/lime-echarts/index`

## 基本用法

```vue
<script setup lang="ts">
import { ref } from 'vue'

const option = ref({
  xAxis: { type: 'category', data: ['Mon', 'Tue', 'Wed'] },
  yAxis: { type: 'value' },
  series: [{ data: [120, 200, 150], type: 'bar' }],
})
</script>

<template>
  <l-echart :option="option" />
</template>
```

## 难点

- 小程序 canvas 层级高，易与 popup/tabbar 冲突，需控制 z-index
- 首次渲染需等容器有宽高，否则图表空白
- 大数据量需开启 dataZoom 或采样

## 易错点

1. **分包路径**：示例在 `pages-demo` 分包，主包引用需注意跨包规则
2. **echarts 体积**：按需引入模块可减小包体（参考 useEcharts.ts）
3. **真机差异**：开发者工具与真机 canvas 表现可能不同，需真机验证

## 注意点

- 也可选用 `ucharts`：`pnpm create unibest add ucharts`
- 静态 echarts 文件在 `uni_modules/lime-echart/static/`

## 适用场景

- 数据看板、统计报表
- 折线/柱状/饼图等常规图表
- 需要全端（H5+小程序+App）一致表现
